# 章节识别逻辑改进总结

## 修改日期
2026-03-26

## 问题背景
原章节识别逻辑过于宽松，导致普通句子（如"这是第一章的内容。"）被误识别为章节标题。

## 核心改进

### 1. 独立成行硬性检查 ✅
**要求**：章节标题必须独立成行，即前面必须有空行（或换行）

**实现**：
```rust
// 硬性检查：章节标题必须独立成行（前面必须有空行）
if line_num > 0 {
    let prev_line = lines[line_num - 1].trim();
    if !prev_line.is_empty() {
        return None; // 前面没有空行，不是章节标题
    }
}
```

### 2. 句子模式硬性检查 ✅
**要求**：以常见句子词开头的文本不应被识别为章节标题

**实现**：
```rust
let sentence_starters = [
    "这", "那", "我", "你", "他", "她", "它",
    "是", "有", "没", "不", "在", "从", "到",
    "因为", "所以", "但是", "而且", "不过",
    "虽然", "尽管", "如果", "假如", "要是",
    "所以", "然后", "接着", "之后",
];

let starts_with_sentence = sentence_starters.iter()
    .any(|starter| trimmed.starts_with(starter));

if starts_with_sentence && !custom_pattern.is_some() {
    return None; // 直接排除
}
```

### 3. 评分权重调整 ✅
**调整**：
- 位置得分权重：0.20 → 0.40（大幅提高）
- 正则得分权重：0.35 → 0.30
- 阈值：0.60 → 0.50

**原因**：独立成行的要求更严格，允许稍微降低总分阈值

### 4. 正则匹配逻辑改进 ✅
**改进前**：部分匹配也给予高分（0.9）
**改进后**：
- 完全匹配前缀：0.9分
- 部分匹配前缀：0.3分
- 内置模式匹配，根据完整性给分（0.2-0.7分）

### 5. 格式得分优化 ✅
**改进**：
- 提高"不包含句号"的权重：0.3 → 0.5
- 提高"不包含过多标点"的权重：0.2 → 0.3
- 降低"包含关键词"的权重：0.4 → 0.2
- 新增对句子结构的检查

### 6. 长度计算修复 ✅
**修复前**：使用字节数 `text.trim().len()`
**修复后**：使用字符数 `text.trim().chars().count()`

**原因**：中文字符在UTF-8中占3字节，导致长度计算错误

## 测试结果

### 改进前的问题
- "这是第一章的内容。" 评分 0.83（误判为章节）
- "这是第二章的内容。" 评分 0.68（误判为章节）
- 普通句子被错误识别，导致章节检测不准确

### 改进后的效果
- "这是第一章的内容。" → 非章节（以"这"开头被排除）
- "这是第二段内容。" → 非章节（以"这"开头被排除）
- "第一章 开始" → 章节（前面有空行，符合格式）
- "第二章 结束" → 章节（前面有空行，符合格式）
